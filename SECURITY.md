# Security Policy

## Supported Versions

This repository is maintained on the `master` branch. Security fixes are applied to the latest code.

## Reporting a Vulnerability

Please do not open a public issue for potential security vulnerabilities.

Report privately through one of the following:

- GitHub Security Advisory (preferred): [https://github.com/Salkimmich/HMeta-d/security/advisories/new](https://github.com/Salkimmich/HMeta-d/security/advisories/new)
- Email the maintainer if advisory tooling is unavailable.

When reporting, include:

- affected file(s) and component(s)
- reproduction steps
- expected vs observed behavior
- impact assessment

## Response Goals

- Initial acknowledgment within 7 days.
- Status update after triage.
- Fix and disclosure timeline coordinated with reporter.
